## KEY FINDING: HomeKit foreground limitation + the fix

- **Problem:** macOS HomeKit only allows characteristic WRITES while the app is foreground/active (Apple's design; not fixable via entitlement). With a normal window app that's buried behind other windows, each poll's HomeKit write fails → log shows "executor failed (will retry)" → the PollLoop retries (nothing lost) but the lamp doesn't change until the app is active again. Confirmed empirically (command sat queued 40s+ while Finder was frontmost).
- **FIX, PROVEN (spike verdict ACCESSORY_WORKS):** set the app's AppKit activation policy to **`.accessory`** (NSApplicationActivationPolicyAccessory — no Dock icon, background/menu-bar style). With this, HomeKit writes SUCCEED even while another app (Finder) is frontmost. Verified via the `homed` daemon log: HAP pair-verify + characteristic PUT returned HTTP 204; two commands drained + acked while Finder stayed frontmost.
- Catalyst can't call `NSApplication.setActivationPolicy` directly. The working技 technique (currently in the uncommitted spike) reaches it at launch via `dlsym`/`objc_msgSend`:

```swift
import Foundation
// Force AppKit activation policy to .accessory (no Dock icon) at launch.
private func setAccessoryActivationPolicy() {
    guard let msgSend = dlsym(UnsafeMutableRawPointer(bitPattern: -2), "objc_msgSend") else { return }
    typealias MsgSendInt = @convention(c) (AnyObject, Selector, Int) -> Bool
    guard let appClass = NSClassFromString("NSApplication") as? NSObject.Type else { return }
    let shared = appClass.value(forKey: "sharedApplication") as AnyObject
    let fn = unsafeBitCast(msgSend, to: MsgSendInt.self)
    _ = fn(shared, NSSelectorFromString("setActivationPolicy:"), 1)   // 1 == .accessory
}
// call from LampControllerApp.init()
```

## How to operate things (reference)

- **Build the app (signed):**
  ```
  export DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer
  cd mac-app && xcodegen generate
  xcodebuild -project LampController.xcodeproj -scheme LampController \
    -destination 'platform=macOS,variant=Mac Catalyst' -derivedDataPath build -allowProvisioningUpdates build
  open build/Build/Products/Debug-maccatalyst/LampController.app
  ```
- **App needs BOTH entitlements** (in `mac-app/Sources/Entitlements.entitlements`): `com.apple.developer.homekit` AND `com.apple.security.network.client` (without the latter, the poll HTTPS is silently blocked — that bit us). App ID `com.volovely.lamp-controller` has the HomeKit capability enabled in the portal.
- **Send a command (wrangler v3 on this machine — note the flags):**
  ```
  cd worker
  UUID=$(uuidgen); NOW=$(date -u +%Y-%m-%dT%H:%M:%SZ)
  pnpm exec wrangler kv key put --namespace-id a0a047d07dd44a07bb0b103b48c75410 \
    "command:$UUID" "{\"id\":\"$UUID\",\"action\":\"on\",\"brightness\":40,\"color_temp_k\":2700,\"created_at\":\"$NOW\",\"source_msg_id\":\"manual\"}"
  ```
  IMPORTANT: this machine has **wrangler 3.114.17** → use `--namespace-id <id>` and do NOT use `--remote` (v3 rejects it). The `worker/wrangler.toml` on `main` still has the `REPLACE_WITH_KV_NAMESPACE_ID` placeholder, so target the id directly. KV namespace id: `a0a047d07dd44a07bb0b103b48c75410` (title `lamp-controller-COMMANDS`).
- **Worker URL:** https://lamp-controller.vistauser1990.workers.dev (`/health` is open; `/commands`+`/ack` need the bearer).
- **Live config:** `~/.config/lamp-agent/config.toml` — command_source=worker, worker_url, shared_secret (matches Worker MAC_SHARED_SECRET), state_path, poll_interval_s=12, homekit_accessory_name="Mijia desk lamp 1S". Mode 0600.
- **Tests:** `cd mac-agent && DEVELOPER_DIR=… swift test` (59), `cd worker && pnpm test` (26), app: xcodebuild test (3).
