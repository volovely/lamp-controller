# Stage 1.5 addendum — HomeKit control via the `shortcuts` CLI

> Addendum to [`2026-05-29-stage-1-mac-to-lamp.md`](2026-05-29-stage-1-mac-to-lamp.md). Same branch (`stage-1-mac-to-lamp`, PR #3). Execute with subagent-driven-development.

## Why this exists

Stage 1 was built against a Homebridge REST contract on the assumption the Mijia lamp would be bridged via `homebridge-miot`. Discovery with the user corrected that:

- The lamp is a **1st-gen Mi LED Desk Lamp (MJTD01YL)**, already in **Apple Home**, and the user wants to **avoid the Xiaomi cloud**.
- That model has **no native HomeKit** and its **Yeelight LAN control was removed** in firmware — so Homebridge/`homebridge-miot` is the wrong path.
- Confirmed working alternative: drive **Apple Home directly via the macOS `shortcuts` CLI** (`shortcuts run "Lamp On"` works; the agent's Mac stays logged into the same Apple ID, which Shortcuts requires).
- Apple's Home action can't bind a **variable** brightness, so we use **preset shortcuts** (fixed brightness levels + Warm/Neutral/Cool) and the agent snaps to the nearest.

The Stage 1 pipeline (CommandSource → PollLoop → CommandExecutor → ack/stale logic) is unchanged. Only the lamp-control client changes. `HomebridgeClient` is retained as an alternative backend but is no longer the default.

## Preset shortcut convention (user creates these once)

Default prefix `Lamp`. Names the `ShortcutsClient` calls:

- Power: `Lamp On`, `Lamp Off`
- Brightness: `Lamp 25`, `Lamp 50`, `Lamp 75`, `Lamp 100` (each sets On + that brightness, touching only brightness)
- Color temp: `Lamp Warm`, `Lamp Neutral`, `Lamp Cool` (each touches only color temperature)

Agent mapping: brightness → nearest of {25,50,75,100}; `color_temp_k` → Warm (≤3300 K) / Neutral (3301–4800) / Cool (>4800).

## Design

**`LampClient`** — the existing 3-closure interface (`setPower`/`setBrightness`/`setColorTemperature`), generalized from `HomebridgeClient`. Registered as `@Dependency(\.lampClient)`. Two factories:
- `LampClient.homebridge(baseURL:token:accessoryId:session:)` — the existing live REST impl (kept, not default).
- `LampClient.shortcuts(prefix:runner:)` — runs preset shortcuts via an injectable `ShortcutRunner`.

**`ShortcutRunner`** — injectable `@Sendable (_ name: String) async throws -> Void`; live impl runs `/usr/bin/shortcuts run "<name>"` via `Process`, throwing on non-zero exit. Tests inject a recording stub.

**Config** — `homebridge_*` fields become optional; add `lamp_backend` (`"shortcuts"` default | `"homebridge"`) and optional `shortcut_prefix` (default `Lamp`). Required fields reduce to `commands_path`, `poll_interval_s`.
